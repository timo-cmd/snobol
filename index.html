SNOBOL 5

What is SNOBOL?

SNOBOL ("StriNg Oriented and symBOlic Language") stands apart from most programming languages of its era by having patterns as a first-class data type (i.e. a data type whose values can be manipulated in all ways permitted to any other data type in the programming language) and by providing operators for pattern concatenation and alternation. SNOBOL4 patterns are a type of object and admit various manipulations, much like later object-oriented languages such as JavaScript whose patterns are known as regular expressions. 

begin
  i = 1
  loop    
      fizzbuzz = ""
      eq(remdr(i, 3), 0)              
      :f(try_5)
      fizzbuzz = fizzbuzz "fizz"
  
  try_5   
      eq(remdr(i, 5), 0)  
      :f(do_mum)
      fizzbuzz = fizzbuzz "buzz"      
  
  do-num  
      ident(fizzbuzz, "")             
      f(show)
      fizzbuzz = i  
      output = fizzbuzz
      i = i + 1
      le(i, 100)                      
      :s(loop)
end
In addition SNOBOL4 strings generated during execution can be treated as programs and either interpreted or compiled and executed (as in the eval function of other languages).

For more reference about SNOBOL4 please visit: wikipedia.org
Why working on a new version?

After SNOBOL4 released, it could to show off and compete with other old languages. It's syntax and sematics seamed to be literally a blast. SNOBOL4 was the first language that introduced string-based patterns and first-class data-types. However, SNOBOL lost its popularity (which is really sad), after Perl and ADA got its first shout-out.

So finally we are working on SNOBOL5, a stable reimplementation of SNOBOL4. The whole compiler was rewritten and redesigned. Until today SNOBOL5 covers a lot of new useful features and is actively maintained.

Abstract SNOBOL5:

SNOBOL version 5 is a dialect of SIL (Snobol Implementation Language). It does inherit directly fro SNOBOL4 and SPITBOL. SNOBOL5 is a high-level, imperative programming language featuring string-scanning and data-oriented programming. It's performance is systematically improved. SNOBOL5 stores variables, strings and data structures in multiple garbage-collected heaps.
